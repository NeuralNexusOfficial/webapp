2. Proposed Tech Stack (for feedback) 
From a tech and scalability perspective, here’s what I recommend: 
Frontend / Full‑ stack Framework 
 Next.js (React + TypeScript) 
o Modern standard for React apps, good SEO, supports mixed SSR/SSG, and ships with API 
routes so we don’t need a separate backend repo.  
Backend / Database / Storage / Auth 
 Supabase (Postgres + Auth + Storage) 
o Managed Postgres with connection pooling, read replicas and production-ready scaling 
patterns, so it can comfortably handle large user bases when needed.

3. High‑ Level Architecture 
Client (browser) 
 Next.js routes for: Home, About, Tracks, Prizes, FAQs, Auth, Dashboard, Admin. 
 Uses Supabase client for auth/DB, and Next.js API routes for payments/admin.  
Next.js API Routes 
 /api/razorpay/create-order → creates Razorpay order. 
 /api/razorpay/webhook → verifies payment signature, updates payments and team/user status to 
PAID.[15][13] 
 /api/admin/... → secure endpoints for admin (scores, leaderboard export etc.). 
Supabase

 Postgres tables: users, teams, team_members, payments, submissions, scores. 
 Auth: user accounts, sessions, row-level security. 
 Storage: PPT/docs uploads for submissions.  
Razorpay 
 Payment gateway, with support for USD and other international currencies once international 
payments are enabled for the account.  
 
4. Product Flow → Features 
Short version of what we will build, mapped to the earlier flow: 
 Phase 1: Landing – Home, About, Tracks, Prizes, FAQs, with “Register Now” and “View 
Tracks”. 
 Phase 2: Registration – Signup/login with email + password (Google later); profile fields 
(Name, Email, Phone, College/Company). 
 Phase 3: Team Formation – Solo participation, or create/join team via invite code + email 
invites. 
 Phase 4: Payments – Show fee per person/team in USD, integrate Razorpay for payment and 
webhook verification.  
 Phase 5: Confirmation & Dashboard – Confirmation email, dashboard with track, team, 
payment and submission status. 
 Phase 6–7: Submission – Resources page + submission form (title, description, track, repo, 
demo, PPT/docs upload). 
 Phase 8: Admin Review – Admin dashboard with filters, scoring (innovation, technical, 
UI/UX, scalability) and comments. 
 Phase 9: Results – Public leaderboard, winner emails, optional live event embed. 
Status lifecycle we’ll respect in the data model: 
REGISTERED → TEAM_CREATED → PAID → SUBMITTED → UNDER_REVIEW → SHORTLISTED → WINNER.
